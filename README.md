# 記銘帖 — 小説アーカイブサイト

## はじめに（初回だけ）

1. [Node.js](https://nodejs.org/) をパソコンにインストールする（まだの場合）
2. このフォルダをターミナルで開いて、次を実行する

```
npm install
```

## 手元でプレビューする

```
npm start
```

ターミナルに表示される `http://localhost:8080` のようなURLをブラウザで開くと確認できます。
ファイルを保存すると自動的に更新されます。終了するときは `Ctrl + C`。

## 小説を追加する

`src/novels/` フォルダに、新しいテキストファイルを1つ作る（拡張子は `.md`）。
ファイル名は何でもいいですが、英数字にしておくのがおすすめ（例: `nazuke.md`）。

中身はこの形式でコピー＆ペーストして書き換える：

```
---
layout: novel.njk
title: ここにタイトル
date: 2026-07-18
type: 短編
fandom: ここに原作名
pairing: ["カップリング名"]
rating: 全年齢
---

ここから本文を書く。
```

### rating（年齢指定）の書き方
- 全年齢の場合: `rating: 全年齢`（省略も可）
- R-18の場合: `rating: R-18`
- R-18Gの場合: `rating: R-18G`

R-18 / R-18G を指定すると、ページを開いたときに自動で警告文＋「同意して読む」ボタンが表示されます。

### pairingが複数・なしの場合
```
pairing: ["CP1", "CP2"]
```
カップリングがない作品は `pairing:` の行ごと消してOK。

## ブログを追加する

`src/blog/` フォルダに新しい `.md` ファイルを作る：

```
---
layout: post.njk
title: ここにタイトル
date: 2026-07-18
excerpt: 一覧ページに表示される一言（省略可）
---

ここから本文を書く。
```

## 大量の作品を一括で流し込む

1本ずつ `.md` を手打ちしなくても、Excelとテキストファイルから一括生成できます。

1. `imports/list.xlsx` を開き、「作品リスト」シートに1行＝1作品で入力する（詳しい書き方は「使い方」シートを参照）
2. 本文は `imports/texts/` フォルダに、ファイル名列と同じ名前の `.txt` ファイルとして置く（例: ファイル名が `001` なら `imports/texts/001.txt`）
3. ターミナルで次を実行する

```
python scripts/generate_novels.py
```

`src/novels/` フォルダに `.md` ファイルがまとめて作られます。エラーがあれば行番号つきで表示されるので、それを直してもう一度実行すればOK（既存のファイルは名前が同じなら上書きされます）。

※ Pythonが入っていない場合は [python.org](https://www.python.org/) からインストールしてください。
初回だけ次を実行して、必要なパッケージを入れます。

```
pip install -r requirements.txt
```

## 公開する（GitHub Pages）

1. GitHubでアカウントを作り、新しいリポジトリを作成する
2. このフォルダの中身（`node_modules` は除く）をリポジトリにアップロードする
3. リポジトリの Settings → Pages で、GitHub Actions を使ったEleventyのデプロイを設定する

このステップは実際にやるときに、一つずつ画面を確認しながら一緒に進めましょう。

## フォルダ構成メモ

- `src/novels/` … 小説
- `src/blog/` … ブログ記事
- `src/assets/style.css` … デザイン（色・フォントなど）
- `.eleventy.js` … サイトの仕組みの設定
